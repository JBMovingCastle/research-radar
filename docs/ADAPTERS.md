# 适配器说明

所有采集器返回统一字段和独立状态。主流程不会因为一个渠道失败而终止。

| 适配器 | 默认 | 密钥 | 主要能力 | 典型限制 |
| --- | --- | --- | --- | --- |
| Crossref | 开启 | 无 | 关键词、日期、期刊、ISSN、DOI | 摘要覆盖不完整 |
| arXiv | 开启 | 无 | 关键词、分类、预印本摘要/PDF | 建筑管理类覆盖有限 |
| Semantic Scholar Search | 开启 | 可选 | 跨学科搜索、引用、开放链接 | 无密钥用户共享限流 |
| Semantic Scholar Recommendations | 开启 | 可选 | 正负种子论文推荐 | 无正向种子时跳过 |
| OpenAlex | 关闭 | 必需 | 关键词、作者、来源、开放链接 | API key与每日额度 |
| RSS/Atom | 无配置 | 无 | 期刊/机构订阅 | 取决于订阅质量 |
| Official Web Page | CI3模板开启 | 无 | 大学、政府、行业网页线索 | 只做链接级发现 |

状态含义：

- `success`：请求完成，不代表一定有入选内容。
- `partial`：同类多个自定义渠道中有成功也有失败。
- `limited`：接口返回限流。
- `failed`：网络、格式或服务错误。
- `misconfigured`：已启用但缺少必需配置。
- `disabled`：用户关闭。
- `skipped`：当前无种子或无自定义来源，因此没有请求。

适配器统一设置超时、有限重试和 `Retry-After` 上限。不会为追求“抓全”进行无限翻页或无限重试。
